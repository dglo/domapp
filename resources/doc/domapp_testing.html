Testing DOMApp


The DOMApp test script is called domapp_multitest.pl.  It uses a C language messaging program called domapptest.  These programs live in the domhub-tools project. 

Example:
[jacobsen@drivertest domapp]$  $tools/domapp/domapp_multitest.pl -h
Usage: domapp_multitest.pl [options]   ...

    DOMs can be "all" or, e.g., "01a, 10b, 31a"
    Must power on and be in iceboot first.

Options:
    -u <img>:    Upload <img? rather testing flash image
    -s:          Show commands issued to domapptest
    -d:          Detailed report about what worked, instead
                 of just what didn't work.
    -F:          Run flasher tests (SEALED, DARK DOMs ONLY)
    -V:          Run tests requiring HV (SEALED, DARK DOMs ONLY)
    -l :   Load FPGA image  from flash before test

If -V or -F options are not given, only tests appropriate for a
bare DOM mainboard are given.
