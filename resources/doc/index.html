MAJOR NEW FEATURES

  1. Hit buffering and Road Grader data compression are now carried out in the DOM FPGA, resulting in substantially less dead time; software interface to these features is in place (including automated calculation of average pedestal patterns in DOM);

  2. Supernova rate meter system implemented;

  3. "Heartbeat" or "beacon" signal of forced-trigger hits generated during normal operation per requirements of higher-level DAQ software components;

  4. Full-second SPE / MPE scaler rate meters in monitoring stream;

  5. Set/query several new local coincidence parameters;

  6. Much more complete (compared to testdomapp) testing infrastructure (domapp_multitest.pl);

  7. In the porting process from testdomapp, the build system and code has been cleaned up;

  8. Nearly nonexistent testdomapp documentation has been filled out into a full API document for domapp.


DOCUMENTATION

DOMApp messaging API is available here.  This describes the messaging interface to domapp, as well as salient differences between domapp and testdomapp.

The DOMApp Test Script is now documented as well (work in progress).

OUTSTANDING ISSUES

  • A few things are not thoroughly tested yet, notably LC functionality, flashers, and programmable dead time. 
  • Compressed data has not been thoroughly examined.
  • A few key messages still missing from the documentation.
  • Request has been made for a rework of the monitoring to reduce overall bandwidth and increase relevance of monitoring information.
  • Should implement raw data format so that compressed data can be compared with uncompressed in a one-to-one format.
